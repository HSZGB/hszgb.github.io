Git学习笔记

发表于2024-08-21|更新于2024-08-21
|阅读量:

参考

https://learngitbranching.js.org/

基础

git init：初始化

git add <filename>：提交文件到暂存区
git add -A，提交所有变化

git commit -m "注释"，

git status 查看状态
--short 或 -s 可以以更简洁的方式呈现

git diff <filename>：工作目录中的文件与暂存区文件的差异
--staged：已暂存文件和最后一次提交文件的差异

.gitignore

可以让某些文件不让git管理，就是也不会让它们出现在untrack files里。

列举暂存区文件

git ls-files

移除文件

git rm <file>：将文件从暂存区移除。

然而刚添加到暂存区的或已在暂存区中被修改的文件不会被删除，防止误删未添加到快照的数据。
选项-f可以强制删除。

如果直接用rm <file>，只会把文件从目录中删除，然而在暂存区中它还是存在，还要执行git add把这次删除记录提交到暂存区里。
然而用git rm可以让其变简洁。

将文件从git中删除，而还保存在工作目录中

即把文件保存在磁盘中，但不想被git继续跟踪。
git rm --cached <file>

分支

git branch：列出所有分支，并用*标出当前所在分支（当前HEAD指针指向的分支）。

git branch <name>：新建一个分支。
-d：删除这个分支。

git checkout <name>：切换分支
-b选项可以新建并切换到这个新建的分支上。

git merge <name>：将目标分支合并到当前分支上。

遇到冲突时的分支合并

比如两个分支都对同一个文件进行了修改，这时候合并会有冲突。
git会合并没用冲突的部分，然而并未做一次提交，等待我们手动解决冲突。

可以手动打开文件进行修改，然后用git add提交每个文件表示冲突已解决。
之后git commit进行提交。

远程仓库

git clone <url>：克隆远程仓库。
此时会在本地仓库新建一个远程仓库分支<remote>/<branch>，像常见的origin/main。
这个分支不会被更新，只有远程仓库相应分支更新时才会更新。
git remote add <name> <url>：链接远程仓库。命名远程仓库，远程仓库链接。
git remote -v：列出远程仓库

git fetch <remote>：拉取远程仓库的更新信息，不会改变工作目录的内容（不会改变main分支），更新远程分支。
origin/main(<remote>/<branch>)是远程分支，指向远程仓库最后一次提交，main指向本地仓库最后一次提交（不改变）。

git push -u 仓库名称 分支 推送到远程仓库
？？没看懂这个

git push <remote> <place>：更新远程仓库的一个分支。
比如说git push origin main：即切到本地仓库中的“main”分支，获取所有的提交，再到远程仓库“origin”中找到“main”分支，将远程仓库中没有的提交记录都添加上去，搞定之后告诉我。
git push <remote> <source>:<destination>：将本地的分支推送到远程仓库的分支。这两个参数分别代表其位置（可以是分支，或者HEAD^之类的）。

git pull：相当于git fetch; git merge

主分支被锁定了

如果远程仓库设置了不能直接commit到main上，然而不小心commit，怎么办。

remote:C0->C1(*main)

local:C0->C1->C2(*main)

应该创建一个新分支feature，然后把C2这次提交放到feature上。再将main重置到C1上。

git branch feature: C0->C1->C2(*main,feature)
git reset C1: C0->C1(*main)->C2(feature)
git checkout feature; git push

变基

git rebase <branch>：将当前分支接到目标分支后。

远程跟踪

当我们push时，main会被推送到远程仓库的main上，同时更新origin/main，这是远程跟踪确定的。

克隆时，git会为远程仓库的每个分支在本地仓库创建一个远程分支，比如(origin/main)，然后再创建一个跟踪远程仓库中活动分支的本地分支，默认情况下这个本地分支会被命名为main。

有两种方法设置这个属性，第一种就是通过远程分支切换到一个新的分支，执行:
git checkout -b totallyNotMain o/main
就可以创建一个名为 totallyNotMain 的分支，它跟踪远程分支 o/main。

另一种设置远程追踪分支的方法就是使用：git branch -u 命令，执行：
git branch -u o/main foo
这样 foo 就会跟踪 o/main 了。如果当前就在 foo 分支上, 还可以省略 foo：
git branch -u o/main

重命名

当我重命名：1.txt->2.txt
它先会显示delete 1.txt，然后untrack 2.txt
然而我git add后，它会识别出rename:1.txt->2.txt

可以直接用git mv <file1> <file2>来做到。

-p --patch显示每次提交带来的差异
-数字 显示最近的几次提交
--stat 每次提交的简略统计信息
--pretty
-- <file>显示某个文件的提交记录
